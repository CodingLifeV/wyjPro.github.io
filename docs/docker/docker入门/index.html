  • 如果项目部署在您的电脑上，无需使用 Docker
  • 仅仅想快速学习如何掌握使用 Docker 部署，学习使用标注🚩的章节

Docker 概述📃

Docker是什么1

Docker 是一个虚拟环境容器，可以将你的开发环境、代码、配置文件等一并打包到这个容器中，并发布和应用到任意平台中。

Info

例如，你在本地用 Java 开发项目，开发测试完成后，就可以将 Java 项目及其依赖包、各种插件、MySQL、Nginx 等打包到一个容器中，然后部署到任意你想部署到的环境。

Docker的三个概念2

  1. 镜像（Image）：类似于虚拟机中的镜像，是一个包含有文件系统的面向 Docker 引擎的只读模板。可以简单理解为一个镜像即是一个可执行程序。

  2. 容器（Container）：类似于运行起来的进程，可以将其看作一个极简的 Linux 系统环境以及运行在其中的应用程序。Docker 引擎利用容器来运行、隔离各个应用。容器是镜像创建的应用实例，如果说 Image 是定义的一个类，那么 Container 则是这个类 new 出来的实例。可以创建、启动、停止、删除容器，各个容器之间是是相互隔离的，互不影响。

    注意

    镜像本身是只读的，容器从镜像启动时，Docker 在镜像的上层创建一个可写层，镜像本身不变。

  3. 仓库（Repository）：类似于代码仓库，这里是镜像仓库，是 Docker 用来集中存放镜像文件的地方。

    注意与注册服务器（Registry）的区别：

    注册服务器是存放仓库的地方，一般会有多个仓库；而仓库是存放镜像的地方，一般每个仓库存放一类镜像，每个镜像利用 tag 进行区分，比如 Ubuntu 仓库存放有多个版本（12.04、14.04等）的 Ubuntu 镜像。

Docker 安装📃

Linux 安装

以 CentOS 为例，要在 CentOS 上开始使用 Docker 引擎，请确保满足先决条件：CentOS 版本要求 CentOS 7 或者 CentOS 8，然后在安装 Docker，更多安装方式可移步官方参考安装教程

安装步骤如下：

  1. 确定 CentOS 版本，要求CentOS 7 或者 CentOS 8

    cat /etc/os-release

    提示

    以下命令都在 Linux 终端执行

  2. 卸载旧版本 Docker

    sudo yum remove docker \
                  docker-client \
                  docker-client-latest \
                  docker-common \
                  docker-latest \
                  docker-latest-logrotate \
                  docker-logrotate \
                  docker-engine

    提示

    旧版本的 Docker 被称为 docker 或 docker-engine，卸载时路径/var/lib/docker/ 下的内容，包括镜像images、容器 containers、卷 volumes 和网络 networks，都将被保留。 Docker 引擎包现在称为docker-ce

  3. 安装yum-utils软件包

    sudo yum install -y yum-utils

  4. 设置镜像的仓库，推荐使用国内阿里云仓库

    # 国内
sudo yum-config-manager \
	--add-repo \    
	https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.rep
   	
# 国外
sudo yum-config-manager \
    --add-repo \
    https://download.docker.com/linux/centos/docker-ce.repo

  5. 安装 Docker 社区版

    sudo yum install docker-ce-<VERSION_STRING> docker-ce-cli-<VERSION_STRING> containerd.io

    提示

    1. docker-ce为社区版

    2. <VERSION_STRING>为版本号，例如docker-ce-18.09.1，此命令会安装 Docker，但不会启动 Docker。 它还创建了一个 Docker 组，但是默认情况下它不会将任何用户添加到该组中。 <VERSION_STRING>为可选项，也可以直接省略安装：

      sudo yum install docker-ce docker-ce-cli containerd.io

  6. 测试是否安装成功

    sudo docker version

    安装成功

  7. 启动 Docker

    sudo systemctl start docker

  8. 通过运行 hello-world 镜像来验证 Docker 引擎是否已正确安装

    sudo docker run hello-world

    提示

    此命令下载测试映像并在容器中运行它，当容器运行时，它会打印一条消息并退出。

Docker 命令📃

帮助命令

查看Docker版本信息

docker version

查看Docker系统信息

docker info

帮助命令

docker [命令COMMAND] –help

Tip

每一个命令大括号[...]中的内容为可选参数，例如

[root@VM-24-11-centos ~]# docker version --help
Usage:  docker version [OPTIONS]
Show the Docker version information
Options:
  -f, --format string       Format the output using the given Go template
      --kubeconfig string   Kubernetes config file
[root@VM-24-11-centos ~]#

打印出了docker version命令的使用方式，使用docker --help可以打印出所有命令的简单

  • docker version：查看 Docker 版本信息

    docker version

    Success

  • docker info：查看 Docker 系统信息，包括镜像和容器的数量

    docker info

    Success

  • docker [命令COMMAND] --help：查看 Docker 系统信息，包括镜像和容器的数量

    docker --help

    Success

镜像命令

查看本地主机镜像信息

docker image

查看存在的镜像信息

docker search

下载镜像

docker pull

删除镜像

docker rmi -f < IMAGE ID>

  • docker image：查看所有本地主机镜像信息

    [root@VM-24-11-centos ~]# docker images
REPOSITORY            TAG          IMAGE ID       CREATED        SIZE
docsy/docsy-example   latest       3654c1a49620   2 days ago     673MB
klakegg/hugo          ext-alpine   99297968b5d5   4 weeks ago    671MB
hello-world           latest       feb5d9fea6a5   3 months ago   13.3kB

    • REPOSITORY：镜像的仓库源

    • TAG：镜像标签

    • IMAGE ID：镜像ID

    • SIZE：镜像大小

    • CREATED：镜像创建时间

  • docker search：查看存在的镜像信息

    docker search hello-world

    Success

  • docker pull：下载镜像

    docker pull < IMAGE ID> # 默认下载最近 latest 版本的镜像
docker pull  < IMAGE ID>[: tag] # 下载指定版本的镜像

    docker pull kitematic/hello-world-nginx

    Success

  • docker rmi -f <IMAGE ID> [IMAGE ID] [IMAGE ID]：删除镜像

    docker rmi -f 3654c1a49620

    Tip

容器命令

新建容器并运行

docker run

查看容器信息

docker ps [OPTIONS]

删除容器

docker rm

查看本地系统镜像信息

docker start

查看存在的镜像信息

docker restart

下载镜像

docker stop

下载镜像

docker kill

  • docker run：新建容器并运行

    docker run [可选参数] 镜像IMAGE [COMMAND]

    常用可选参数：

    • --name="NAME"：容器名字
    • -d：容器在后台运行
    • -it：以交互模式运行容器，并为容器重新分配一个伪输入终端，使用命令exit退出容器
    # 启动容器, 进入容器环境(伪输入终端),类似一个极简的Linux环境,可以执行Linux相关命令
[root@VM-24-11-centos ~]# docker run -it centos:latest /bin/bash

    Success

    # 退出容器
[root@3a6349d6c1b4 /]# exit 
exit

    Tip

    1. 使用exit退出容器，此时容器的状态处于 Exit，而不是后台运行。若不使用 exit，而是使用CTRL+p CTRL+q 退出，此时容器的状态为Up

    2. 如果使用-d参数，则只会在后台运行容器centos:latest，而不会进入伪终端

  • docker ps：显示当前正在运行的容器

    可选参数：

    • -a：显示所有的容器，包括未运行的
    • -n：列出最近创建的n个容器
    • -q：只显示容器的编号
    # docker ps -a

    Tip

  • docker rm：删除容器

    docker rm 容器ID     # 删除指定容器, 不能删除正在运行的容器
docker rm -f 容器ID  # 强制删除指定容器
docker rm $(docker ps -a -q) # 删除所有已经停止的容器

    Tip
